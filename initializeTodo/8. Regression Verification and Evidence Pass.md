# 8. Regression Verification and Evidence Pass

Status: Not Started  
Mandatory: Yes  
Depends On: Tasks 0-7

## Goal

Run a final evidence pass after the full mock is implemented, verify the Next.js build, and make sure the todo files accurately describe what was completed.

## Verification Scope

- Confirm all numbered tasks have completion evidence.
- Confirm `initializeTodo/taskTracker.md` statuses match the numbered files.
- Confirm no `page.tsx` contains direct section implementation markup.
- Confirm public route files are under `app/(public)/`.
- Confirm no generated shadcn file in `components/ui/` was modified without an explicit reason.
- Confirm all `[PLACEHOLDER]` and `[TODO]` markers are intentional and searchable.
- Confirm copied assets are used from `public/images/jodi/`.
- Confirm no Create Next App starter UI remains on `/`.

## Commands

Run:

```bash
npm run lint
npm run build
```

If a dev server is needed for visual checks, run:

```bash
npm run dev
```

## Manual Verification

- Desktop viewport around 1440px.
- Tablet viewport around 768px.
- Mobile viewport around 390px.
- Header sticky behavior.
- Mobile menu open/close.
- Anchor navigation to all sections.
- Form keyboard navigation.
- Footer link visibility.
- No visible text overlap.

## Acceptance Criteria

- Build passes.
- Lint passes or any failure is documented with exact output and cause.
- Responsive manual checks are recorded.
- Evidence blocks in all task files are populated.
- Tracker has accurate final statuses.

# 7. Footer, Metadata, SEO, and Polish

Status: Not Started  
Mandatory: Yes  
Depends On: Tasks 1-6

## Goal

Finish the public shell with the footer, metadata, accessibility details, and visual polish needed before final verification.

## Reference Mock Findings

- Footer repeats the brand, short description, social links, three link groups, legal links, and copyright.
- Footer columns are Shop, Train, and Studio.
- The mock includes social icons for Instagram, TikTok, YouTube, and Spotify.
- Metadata title in the mock is `Jodi Stokes Fitness - Train. Lifestyle. Nutrition.`

## Implementation Scope

- Create `components/section/shared/site-footer.tsx`.
- Add footer to the public layout or home composition.
- Update `app/layout.tsx` or route metadata for:
  - title
  - description
  - Open Graph basics if appropriate
  - viewport/theme color if useful
- Ensure all hash links resolve to existing IDs.
- Add focus-visible states for interactive elements.
- Review spacing, contrast, hover states, and mobile stacking.
- Mark social and legal links with `[PLACEHOLDER]` if URLs are fake.
- Mark future real policy/social URLs with `[TODO]`.

## Files To Inspect

- `reference-html/index.html` Footer section.
- `app/layout.tsx`
- `app/(public)/page.tsx`
- `components/section/shared/`

## Acceptance Criteria

- Footer visually matches the mock and is responsive.
- Metadata no longer uses Create Next App defaults.
- Placeholder links/actions are explicitly marked in code.
- Keyboard focus is visible across nav, cards, forms, and footer.
- Text does not overlap or overflow on common mobile widths.

## Tests

- Run `npm run lint`.
- Run `npm run build`.
- Manual keyboard tab pass through header, forms, cards, and footer.
- Manual mobile check at 390px width.

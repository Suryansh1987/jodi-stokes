# 1. Public Layout, Header, Mobile Navigation

Status: Not Started  
Mandatory: Yes  
Depends On: Task 0

## Goal

Recreate the mock topbar, sticky navigation, brand mark, desktop links, action buttons, and mobile menu as shared public layout components.

## Reference Mock Findings

- The topbar is a mint marquee with repeated promotional messages.
- The nav is sticky, translucent black, blurred, and bordered.
- Brand shows circular `JS` mark plus `JODI STOKES` and small service tagline.
- Desktop links anchor to `about`, `programs`, `book`, `shop`, `journal`, and `coaching`.
- Actions include search, cart with a `3` badge, and `Book a call`.
- Mobile replaces links with a hamburger and fullscreen dark menu.
- The mock uses inline SVG icons; Next implementation should use `lucide-react` icons where possible.

## Implementation Scope

- Create shared components in `components/section/shared/`:
  - `brand-logo.tsx`
  - `topbar.tsx`
  - `site-header.tsx`
  - `mobile-nav.tsx`
- Add header/nav to the public route layout, either `app/(public)/layout.tsx` or the home composition if a route layout is not yet useful.
- Use `next/link` for route/hash navigation.
- Use `lucide-react` for Search, ShoppingBag, Menu, X, ArrowRight, and social icons when available.
- Make the mobile nav a client component only where interactivity is required.
- Mark fake search and cart behavior with `[PLACEHOLDER]`.
- Mark future cart/search implementation with `[TODO]`.

## Files To Inspect

- `reference-html/index.html` lines around the topbar, nav, and mobile nav.
- `app/layout.tsx`
- `app/(public)/page.tsx`
- `components/ui/card.tsx`

## Acceptance Criteria

- Header matches the mock hierarchy and visual weight.
- Desktop nav is visible at larger widths and hidden on mobile.
- Mobile menu opens/closes, updates `aria-expanded`, and prevents inaccessible hidden content.
- Anchor links target section IDs used by the page sections.
- Header remains sticky without overlapping first viewport content incorrectly.

## Tests

- Run `npm run lint`.
- Run `npm run build`.
- Manual desktop check at 1440px width.
- Manual mobile check at 390px width, including menu open/close.

# 0. Initialize Structure, Globals, Assets

Status: Completed  
Mandatory: Yes  
Depends On: None

## Goal

Prepare the Next.js project structure, global styling foundation, font setup, and public assets needed to rebuild the reference mock.

## Reference Mock Findings

- `reference-html/index.html` defines the full page styles inline.
- Brand colors are dark black surfaces with mint `#14E8B0`, coral `#FF4D5E`, gold `#F5C04A`, bone `#f6f4ee`, and layered white alpha tokens.
- Fonts in the mock are `Syne`, `DM Sans`, and `Caveat`.
- Required core images are:
  - `reference-html/assets/jodi-portrait.jpg`
  - `reference-html/assets/jodi-leadership.png`
  - `reference-html/assets/jodi-plank.png`
  - `reference-html/assets/book-water-exercise.png`

## Implementation Scope

- Read the relevant Next.js docs in `node_modules/next/dist/docs/01-app/`.
- Create `app/(public)/` and move the public home route there.
- Keep `app/page.tsx` removed or converted so only `app/(public)/page.tsx` owns `/`.
- Create section folders under `components/sections/home/`.
- Create shared folder `components/section/shared/`.
- Create `lib/content/`.
- Copy required reference assets into `public/images/jodi/`.
- Update `app/globals.css` with brand variables, Tailwind theme mappings, base body styles, noise overlay if retained, responsive defaults, and shared button/card utilities only when they reduce duplication.
- Update font imports in `app/layout.tsx` with the selected Next font strategy.
- Update metadata title/description away from Create Next App.

## Structural Requirements

- Do not modify `components/ui/` generated files.
- Prefer CSS variables and Tailwind utilities over one-off inline styles.
- Preserve existing shadcn/Tailwind imports in `app/globals.css`.
- Add `[PLACEHOLDER]` comments near any retained temporary mock data.
- Add `[TODO]` comments near future integration points.

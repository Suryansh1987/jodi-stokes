# 2. Home Page Composition and Content Map

Status: Completed
Mandatory: Yes
Depends On: Task 0

## Goal

Create a thin home `page.tsx` that composes section components and centralize repeated mock content in typed data.

## Reference Mock Findings

- The mock is a single long public landing/storefront page.
- Content is repeated across nav, footer, cards, buttons, and forms.
- Several sections use card arrays: pillars, programs, book features, products, testimonials, coaching features, and journal posts.

## Implementation Scope

- Create `app/(public)/page.tsx` that only imports and renders section components.
- Create `lib/content/home.ts` for typed arrays and constants.
- Create shared component `components/section/shared/section-intro.tsx`.
- Define content for:
  - marquee messages
  - nav links
  - hero stats
  - pillars
  - credentials
  - programs
  - book features
  - shop categories/products
  - testimonials
  - coaching features
  - journal posts
  - footer link groups
- Mark mock product/cart/form records with `[PLACEHOLDER]` comments.
- Mark real CMS/ecommerce/form integration with `[TODO]` comments.

## Page Composition Requirement

`app/(public)/page.tsx` should look like composition, not implementation. It should not contain section markup, card markup, hardcoded arrays, or inline CSS.

Expected shape:

```tsx
export default function HomePage() {
  return (
    <>
      <HeroSection />
      <PillarsSection />
      <AboutSection />
      <ProgramsSection />
      <BookSection />
      <ShopSection />
      <TestimonialsSection />
      <CoachingSection />
      <JournalSection />
      <NewsletterSection />
    </>
  );
}
```

## Files To Inspect

- `reference-html/index.html`
- `app/page.tsx`
- `app/(public)/page.tsx`
- `lib/utils.ts`

## Acceptance Criteria

- Home page route is composed from imported sections only.
- Section IDs match nav/footer links.
- Repeated content is not duplicated across components.
- Types make it hard to accidentally omit required card fields.

## Tests

- Run `npm run lint`.
- Run `npm run build`.

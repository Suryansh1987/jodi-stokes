Status: Not Started

## Current Component

## MCP Candidates Searched

| MCP Server | Exact Component Name | Fit |
|---|---|---|
| `magic` | `Contact Form` | Strong fit for the application form surface with labels, inputs, selects, and textarea. |
| `magic` | `Contact 2` | Good split contact fallback, but less tailored to an application workflow. |
| `magic` | `Form` | Generic form fallback; weaker than the contact/application composition. |
| `magicui` | `blur-fade` | Strong fit for staged reveal of cohort details and form. |
| `magicui` | `border-beam` | Possible accent around the application form. |
| `magicui` | `bento-grid` | Too structural; current split panel is already appropriate. |
| `magicui` | `noise-texture-demo-2` | Newsletter-shaped, not suitable for coaching application. |

## Proposed Replacement

- Prefer 21st.dev `Contact Form` for the coaching application form surface.
- Keep the current coaching offer, investment summary, and local-only form behavior.
- Use Magic UI `blur-fade` only as secondary reveal polish after the form structure remains accessible.

## MCP Server Used

- `magic`
- `magicui`

## Exact Component Name From MCP Search

- `Contact Form`

## Why It Was Chosen

The coaching section is closest to an application/contact workflow. 21st.dev `Contact Form` provides a stronger form composition while allowing the current labels, required fields, and placeholder submission behavior to remain explicit.

## Estimated Implementation Complexity

Medium

## Dependencies

- 21st.dev `Contact Form`
- Existing shadcn `Button`, `Card`, `Input`, `Label`, `Select`, and `Textarea`
- Radix Select dependency if the selected component uses shadcn Select

## Are Motion Animations Alone Sufficient?

No. The useful improvement is form composition and application UX, not only entrance animation.

## Acceptance Notes

- Preserve labels and required fields.
- Keep `[PLACEHOLDER]` / `[TODO]` comments for CRM/backend submission.
- Do not animate form fields in a way that harms usability.

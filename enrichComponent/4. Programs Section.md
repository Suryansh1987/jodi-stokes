# 4. Programs Section

Status: Not Started

## Current Component

- `components/sections/home/programs-section.tsx`

## MCP Candidates Searched

| MCP Server | Exact Component Name | Fit |
|---|---|---|
| `magicui` | `magic-card` | Strong fit for premium hover treatment on program cards. |
| `magicui` | `bento-grid` | Good structural candidate, but current three-card hierarchy already works. |
| `magicui` | `border-beam` | Good for the featured program only. |
| `magicui` | `neon-gradient-card` | Rejected; too neon for the current mint/coral editorial brand. |
| `magic` | `Pricing Card` | Strong semantic fit for paid programs with plan name, price, badge, features, and CTA. |
| `magic` | `Pricing Base` | Good fallback but less polished than `Pricing Card`. |

## Proposed Replacement

- Prefer adapting 21st.dev `Pricing Card` for program cards.
- Keep the existing program content, tags, price, featured state, image/art treatment, and visual-only add actions.
- Use Magic UI `magic-card` only as a secondary hover enhancement if the pricing-card structure is not adopted.

## MCP Server Used

## Exact Component Name From MCP Search

## Why It Was Chosen

21st.dev `Pricing Card` is more semantically useful than Magic UI `magic-card` because the section sells paid programs. It already models plan name, badge, price, feature list, and CTA.

## Estimated Implementation Complexity

Medium

## Dependencies

- 21st.dev `Pricing Card`
- Existing shadcn `Button`
- Existing `lucide-react` icons
- Existing shadcn `Card`

## Are Motion Animations Alone Sufficient?

No. The enrichment is mainly a product/pricing card structure change; Motion-only polish is not enough.

## Acceptance Notes

- Featured card must remain visually distinct.
- Buttons remain `[PLACEHOLDER]` until checkout/cart is wired.
- Hover effects must not be the only visible affordance.

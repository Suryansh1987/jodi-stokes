- Prefer 21st.dev `Resizable Navbar` if the future implementation accepts a larger navigation refactor.
- Keep the existing custom topbar behavior or use Magic UI `marquee` only as a secondary enhancement.
- Preserve current cart/search/theme placeholder behavior and route hash links.

## Why It Was Chosen

21st.dev `Resizable Navbar` is a closer complete-section candidate than Magic UI `marquee` because it covers desktop links, sticky scroll treatment, CTA buttons, and mobile navigation. Magic UI `marquee` remains useful only for the announcement strip.

## Estimated Implementation Complexity

High

## Dependencies

- 21st.dev `Resizable Navbar`
- `motion/react`
- `@tabler/icons-react`
- Existing `next/link`, brand logo, theme toggler, and placeholder search/cart actions must be adapted.

## Are Motion Animations Alone Sufficient?

No. This is a full navigation component comparison. Motion helps with scroll and menu animation, but link behavior, placeholders, theme toggle placement, and accessibility must be adapted manually.

## Acceptance Notes

- Header layout and mobile menu accessibility must remain unchanged.
- Announcement text must remain deterministic.
- Do not use `dock` for the public nav.

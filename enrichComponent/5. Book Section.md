## Proposed Replacement

- Prefer adapting 21st.dev `Feature Showcase` if the book section should become a richer product-feature experience.
- Preserve the current 3D book cover, price row, feature list, and book CTAs.
- Use Magic UI `border-beam` only as a lower-risk accent around the existing book cover.

## Why It Was Chosen

21st.dev `Feature Showcase` is a better section-level candidate than Magic UI `border-beam` because the book section is a product feature with media, benefits, and purchase CTAs. It gives more structure than a decorative border effect.

## Estimated Implementation Complexity

High

## Dependencies

- 21st.dev `Feature Showcase`
- shadcn/Radix `tabs`
- shadcn/Radix `accordion`
- shadcn `Button`, `Badge`, `Card`
- Existing `next/image` book cover asset.
- Optional fallback: Magic UI `border-beam` with `motion`.

## Are Motion Animations Alone Sufficient?

No. The preferred candidate needs structural adaptation. Motion alone is only sufficient for the fallback border accent.

## Acceptance Notes
